# 物流/液体/热量/电力的物流传输逻辑

Mindustry中的各种资源从产生到消耗，中间必然会经历传输的过程。很多时候，往往是传输过程才是资源生命周期的重点。本节我们将聚焦原版中五种主要资源中四种的传输过程，第五种载荷将会放到下一节介绍。

原版的五种资源传输方法各有不同，差异巨大，机制都无法互通，因此我们必须分开来讨论。

## 物品传输

在上一节中，我们曾经说过可以通过直接操纵`items`来处理物品相关的逻辑。但对于物流系统来说这一点并不成立。因为直接使用`items`至少有一个问题：对方是否可以接受这种物品？不能接受的原因有很多，例如在不在接受列表`itemFilter`当中，此物品数量已经达到最大容量，等等。但是，这些原因又有各自的特殊情况，例如对于存储类来说，物品容量并不是通过`itemCapacity`定义的，而是`storageCapacity`；分离器是按总物品数量计数的，大分离器又要去除掉废料再计数……从以上可以看出，让物品提供者去手动判定对方是否可以接受物品是不现实的。这时候就需要用到我们之前提到的多态了。
